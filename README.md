## The leverage measure

`L_i = C_i · (V_i / D) · (1 + B_i) · (1 − open_i)` — concentration × versatility-weight ×
(1 + bridge-load) × openness-discount. **要 = argmax L.** Concentration alone is *not* the
bottleneck; versatility (domains spanned) is the system-of-systems discriminator.

## Safety (what makes a "find the leverage point" tool charter-clean)

It is **constitutionally incapable** of producing a kill-list or a seize-plan:

- the 要 is always a **structural position**, never a natural person (G1; person-excluded);
- it can **only** route to OPENING / route-around / redundancy / decentralization — `capture` /
  `seize` / `control` are unrepresentable (G2);
- it **proposes**; ossekai **carries**, consent-bound + transparent (G3);
- ideology/religion appear only as **structural interfaces** with an on-the-record basis, never a
  belief verdict (G5).
